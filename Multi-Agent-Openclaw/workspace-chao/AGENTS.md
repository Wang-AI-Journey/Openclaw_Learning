## 如何给课题组成员发任务
使用 `sessions_send` 工具（同步，会等待回复）：
- 发给林晓杭（杭仔）：`sessionKey` 填 `"agent:hang:chao"`，`timeoutSeconds` 设为 `120`
- 发给杨若云（小云）：`sessionKey` 填 `"agent:yun:chao"`，`timeoutSeconds` 设为 `120`

**重要：不要用 sessions_spawn！** 用 `sessions_send` 可以同步等待对方回复结果。

## 串行协作策略（重要）
当需要多人串行协作时（比如杭仔写代码，小云写科普）：
1. 先用 `sessions_send` 把任务发给第一个人，等待回复
2. 从回复中提取关键内容（代码、数据等）
3. 把提取的内容完整地包含在消息里，发给下一个人

**关键原则：通过消息传递内容，不要让成员去读其他人的文件。**
每个人的 workspace 是隔离的，互相读不到对方的文件。

示例：
- 第一步发给杭仔：`"请写一个水质监测的 Python 脚本"`
- 杭仔回复了代码
- 第二步发给小云：`"请基于以下代码写一篇科普文章：\n\n（这里粘贴杭仔回复的完整代码）"`

## 读取其他成员的群聊历史

必须用的场景：
- 用户说【基于XX刚才说的】【XX写的代码/文章】【你看看XX的内容】
- 你需要引用或评价其他成员的产出，但消息里没有具体内容

固定步骤（不要跳过）：
1. 调用 sessions_history 工具
2. 从返回结果里找到相关内容
3. 基于找到的内容完成任务

sessionKey 对照表（直接复制使用）：
- 林晓杭：agent:hang:feishu:group:YOUR_GROUP_ID
- 杨若云：agent:yun:feishu:group:YOUR_GROUP_ID
- 参数：limit=10，includeTools=false

注意：不要凭空猜测其他成员说了什么，先查再答。
